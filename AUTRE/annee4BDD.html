Cette UAA est très importante car elle sera utilisée aussi bien dans la partie Web que dans la partie programmation. Dans 90% des projets existants, une base de données existe pour alimenter l'application. C'est un élément essentiel et le premier point d'attaque pour les hackers.

  1. Étape 1

    MCD

    1. Tout le vocabulaire lié à la BDD (entités, propriétés, association, ...)
    2. Construire une entité
    3. Ajouter à cette entité ses propriétés
    4. Chaque propriété aura son bon type
    5. Relier ces entités avec une association correct
    6. Ajouter les cardinalités de cette association
    7. Savoir citer chaque cardinalité par son nom
    8. Faire un schéma grâce à draw.io
    9. A partir d'une situation, sortir le schéma MCD
  2. Étape 2

    MLD

    1. Notion de clé étrangère
    2. Transformer chaque entité en table
    3. Passer d'une association many-to-many à une nouvelle table avec la création de la PK et les FK
    4. Passer d'une association one-to-many à une FK ajouté du côté many
    5. Passer d'une association one-to-one à une FK ajouté d'un des deux côté
  3. Étape 3

    Création et insertion

    1. À partir d'un MLD, créer les tables
    2. Ajouter les PK et FK directement ou après la création
    3. Modifier les caractéristiques des données
    4. Insérer des données
    5. Modifier des données
  4. Étape 4

    Récupérer des données

    1. Récupérer une table entière
    2. Ajuster le résultat à certaines colonnes de la table
    3. Limiter le nombre de résultat selon des critères
    4. Utiliser les fonctions liées au langage SQL
    5. Faire des sous-requêtes
    6. Joindre plusieurs tables
